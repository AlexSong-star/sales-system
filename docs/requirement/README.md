## 一、项目概述

### 1.1 项目名称
外贸数字团队 - 销售管理系统

### 1.2 项目背景
为外贸数字团队搭建独立的销售管理系统，覆盖从客户询盘到成交的全流程数字化管理。

### 1.3 项目范围（本期）
**第一期：销售仪表盘 + 询盘统一收口**

### 1.4 远期规划
- [ ] 客户管理（CRM）
- [ ] 订单管理（询盘→成交→交付）
- [ ] 产品管理（商品库、价格表）
- [ ] 供应链/库存
- [ ] 财务报表（营收、成本、利润）
- [ ] 团队协作（任务分配、进度追踪）

## 二、第一期需求说明

### 2.1 核心目标
把三个客户渠道（邮件、WhatsApp、阿里国际站）的询盘数据统一收口，
吕秀才的回复内容可视化跟踪，团队其他成员也能查看使用。

### 2.2 业务渠道（已确认）
| 渠道 | 说明 | 状态 |
|------|------|------|
| 邮件 | Email 询盘 | 吕秀才对接中 |
| WhatsApp | 即时通讯询盘 | 吕秀才对接中 |
| 阿里国际站 | Alibaba.com 平台询盘 | 吕秀才对接中 |

### 2.3 功能需求（已确认）

#### 仪表盘（Dashboard）
- [ ] **总览统计**：今日/本周/本月询盘总量，各渠道分布
- [ ] **询盘趋势**：折线图展示询盘数量随时间变化
- [ ] **渠道占比**：饼图展示各渠道询盘比例
- [ ] **待处理提醒**：超过N小时未回复的询盘高亮显示
- [ ] **转化漏斗**：询盘→已回复→跟进中→成交 各阶段数量

#### 询盘管理列表
- [ ] **全量展示**：所有询盘，支持翻页（每页20条）
- [ ] **渠道筛选**：按邮件/WhatsApp/阿里站筛选
- [ ] **时间筛选**：按日期范围筛选
- [ ] **状态筛选**：待回复/已回复/跟进中/已成交/已无效
- [ ] **关键词搜索**：按客户名/产品/内容搜索
- [ ] **列表字段**：渠道标识、客户名、产品、询盘内容摘要、收到时间、状态、负责人
- [ ] **可展开详情**：点击行展开，显示完整询盘内容和全部回复记录

#### 询盘详情（展开查看）
- [ ] **原始询盘内容**：客户发送的完整消息原文
- [ ] **客户基本信息**：客户名、联系方式、来源国家
- [ ] **产品信息**：询盘涉及的产品
- [ ] **回复记录**：吕秀才的每条回复（内容+时间，可展开查看完整）
- [ ] **状态变更历史**：状态变化时间线
- [ ] **更新时间**：最后更新时间
- [ ] **备注**：可添加内部备注

#### 权限与使用
- [ ] **团队可见**：佟湘玉、李大嘴、郭芙蓉可查看数据
- [ ] **吕秀才录入/更新**：吕秀才负责录入和更新询盘状态
- [ ] **管理员管理**：白展堂负责系统管理

### 2.4 尚未确认的功能
- [ ] 是否需要自动抓取各渠道的询盘？（vs 手动录入）
- [ ] 询盘状态的具体定义和数量
- [ ] 成交的定义（付款？签合同？）
- [ ] 是否需要客户信息（公司名/联系人/联系方式）
- [ ] 是否需要产品信息（询盘涉及的产品）
- [ ] 数据导出功能（Excel）
